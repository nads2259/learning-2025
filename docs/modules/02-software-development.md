# Module 2: Software Development Methodologies

## Week 1: Requirements & Planning
- Translate business needs into user stories and acceptance criteria
- Compare functional vs. non-functional requirements
- Practice backlog grooming and estimation techniques
- Lab: Build a product requirements document and user journey map

## Week 2: Delivery Frameworks
- Study Waterfall vs. Agile mindsets, Scrum roles, and ceremonies
- Explore Kanban metrics, flow efficiency, and WIP limits
- Introduce Lean principles and continuous improvement loops
- Lab: Run a simulated sprint with task board management

## Week 3: Collaboration & Version Control
- Master Git workflows (feature branches, pull requests, code reviews)
- Set up continuous integration checks and branch protection rules
- Implement documentation standards using Markdown and diagrams
- Lab: Collaborate on a mini project through GitHub Issues and Actions

## Week 4: Quality & Change Management
- Implement test strategies (unit, integration, regression)
- Track defects, perform root cause analysis, and manage releases
- Discuss ITIL change processes and deployment playbooks
- Lab: Design a release plan and retrospective template

## Assessments & Deliverables
- Sprint planning presentation
- Version control lab submission
- Module portfolio: release runbook + retrospective analysis
